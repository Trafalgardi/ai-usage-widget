# AI Usage Widget — виджет лимитов для Windows 11

Небольшое всегда-поверх-окон окошко, которое показывает **остатки лимитов** для
Claude Code, Codex CLI и OpenCode.

**Экраны:**
1. **Обзор** — остаток текущей 5-часовой сессии по всем трём сразу.
2. **Claude** — сессия + неделя (и Opus-неделя, если есть) со временем сброса.
3. **Codex** — сессия + неделя со временем сброса, план, доп. лимиты моделей.
4. **OpenCode** — сессия + неделя + месяц (в % и в долларах).

Каждая карточка оформлена в стиле своего продукта: Claude — кремовый фон, засечки
и коралловый акцент; Codex — минималистичный чёрно-белый OpenAI-стиль; OpenCode —
терминал с моноширинным шрифтом и ASCII-прогрессбаром.

## Установка

1. Нужен **Python 3.10+**: https://python.org (при установке отметь *Add to PATH*).
2. В папке виджета:
   ```
   pip install pywebview pystray Pillow
   ```
3. Запуск:
   ```
   python widget.py
   ```
   или двойной клик по **start_widget.vbs** — запустит без чёрного окна консоли.

Окно можно таскать за верхнюю панель. Кнопки: ⟳ обновить, ◉ поверх окон вкл/выкл, ⎯ свернуть в трей, ✕ закрыть.
Позиция окна запоминается в `config.json`.

### Системный трей

При сворачивании в трей виджет продолжает работать в фоне. Наведи на иконку трея — увидишь тултип с процентами остатка сессий Claude и Codex и временем сброса. Правый клик по иконке:
- **Показать** — вернуть окно
- **Обновить** — запросить свежие данные
- **Выход** — полностью закрыть виджет

## Автозапуск при старте Windows

Нажми `Win+R` → `shell:startup` → скопируй туда **ярлык** на `start_widget.vbs`.

## Откуда берутся данные

Виджет ничего не отправляет наружу, кроме запросов к официальным серверам
самих сервисов. Токены читаются из тех же файлов, что используют CLI:

| Сервис | Токен | Эндпоинт |
|---|---|---|
| Claude Code | `~/.claude/.credentials.json` | `api.anthropic.com/api/oauth/usage` |
| Codex CLI | `~/.codex/auth.json` | `chatgpt.com/backend-api/wham/usage` |
| OpenCode | `~/.local/share/opencode/auth.json` | подбирается автоматически (см. ниже) |

Требование: быть залогиненным в каждом CLI (`/login` в Claude Code, `codex login`,
`/connect` в opencode).

### Важно про OpenCode

У OpenCode (планы Zen/Go) на момент написания **нет официального публичного
usage-API** — лимиты видны только в консоли opencode.ai. Виджет пробует несколько
вероятных эндпоинтов с твоим API-ключом; если ни один не отвечает, карточка
покажет подсказку. Как только эндпоинт появится (за этим следят в issue
anomalyco/opencode#31084), просто впиши его в `config.json`:

```json
"opencode": { "usage_endpoint": "https://opencode.ai/..." }
```

Формат ответа парсится гибко (`rolling5h/weekly/monthly`, проценты или доллары).

### Про эндпоинты Claude и Codex

Оба эндпоинта — те же, что используют сами CLI и утилиты вроде CodexBar, но они
недокументированные и могут поменяться. Если карточка вдруг перестала обновляться
после апдейта CLI — напиши, поправим парсер (он и так принимает несколько
вариантов имён полей).

## Настройки (`config.json`)

```json
{
  "refresh_interval_sec": 60,        // как часто опрашивать API
  "window": { "width": 380, "height": 600, "on_top": true }
}
```

## Возможные проблемы

* **HTTP 401/403** — токен истёк. Открой соответствующий CLI (он сам обновит
  токен) или залогинься заново.
* **Пустое окно** — не установлен WebView2 Runtime (на Windows 11 он есть по
  умолчанию; если нет — https://developer.microsoft.com/microsoft-edge/webview2/).
* Красный цвет полоски = остаток ≤ 15 %.
